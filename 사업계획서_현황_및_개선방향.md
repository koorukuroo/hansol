### 핵심 전환 방향

```
AS-IS (수동 대응형, Reactive)          →    TO-BE (예측 주도형, Predictive)
─────────────────────────────────          ─────────────────────────────────
• 전화 기반 수동 주문 (누락률 15%)       • IoT+AI VMI 자동보충 (누락 0%)
• 탱크 잔량 파악 불가                     • 15분 간격 실시간 모니터링
• 경험 기반 수작업 배차 (공차율 30%)      • AI 경로최적화 (공차율 15%↓)
• 사각지대 육안 확인 (접촉사고 12건/년)   • Around-View AI 위험감지 (300ms)
```

## 4. 개선 방향 (우선순위별)

### PRIORITY 1 — 필수 완료 (미완성 시 평가 불가)

#### 4.1 예산 계획 전체 작성 (VI장)
- **현상**: 모든 금액 셀이 빈칸
- **영향**: 기술성평가 "소요예산 적정성" 5점 + 원가계산 단계 영향
- **작성 방향**:
  - 총 사업비 **15~18억원** 규모 편성 (정부 70% = 10.5~12.6억, 민간 30%)
  - SW 개발비: FP 기반 산출. 모듈별 기능점수 × 단가. 약 5~7억원
  - HW 도입비: IoT센서 200대(1억), 카메라+엣지AI 10세트(2.7억), 서버(0.8억) 등 약 4~5억원
  - 인건비: 참여인력 10명 × 월급 × 투입률 × 12개월
  - N/W 구축: 0.3~0.5억원
  - 기타: 정산수수료(190만원), 기술임치비(45만원) **필수 편성**
  - 기획기관 운영비: 정부지원금의 5% 이내
  - 현물 편성: 도입기업 인건비 + 기구축 장비 (민간부담 50% 이내)

- **작성 방향**: 울산TP의 기존 스마트공장/AI 관련 지원사업 수행 이력 기재

#### 4.5 보안대책 이행계획 (V-4장)
- **현상**: "보안조치 이행계획" 섹션 내용 없음
- **작성 방향**:
  - 참여자 보안서약서 징구 + 보안교육 실시(착수 시 1회, 이후 분기 1회)
  - 개발환경 접근통제: VPN + 2FA 인증 + RBAC
  - 데이터 암호화: 전송(TLS 1.3) + 저장(AES-256)
  - 위치정보 보호: 위치정보법 준수, 기사 동의서 확보
  - 개인정보 보호: 최소 수집 원칙, 보관 기한 설정
  - 보안 사고 대응: 사고 발견 24시간 내 보고 체계 + 긴급 차단 절차

#### 4.6 사업실패 시 귀책사유별 조치 (V-2장)

- **작성 방향**:

  - ② 공급기업 귀책: AI 모델 성능 미달, 납기 지연 등 → 기수령 사업비 반환, 하자보수 의무 이행
  - ③ 공동 책임: 비율 산정 후 각자 분담금 반환, 기구축 부분은 활용 계획 수립

### PRIORITY 2 — 감점 방지 (미완성 시 해당 항목 감점)

#### 4.7 참여인력 상세 정보 (III-3장)
- **현상**: 이름 "OO" 마스킹, 학위/경력/수행내용 빈칸 다수
- **작성 방향**:
  - 총괄책임자(이상훈): 학위, 경력 20년 이상, AI/스마트공장 관련 경력 기재
  - 구축·공급책임자(조규형): **본 과제** 내 역할 구체화 (현재 "디지털클러스터 정부사업 추진"으로만 기재)
  - 나머지 참여인력: 실명, 학위, 전문분야, 투입률 기재

#### 4.8 성과지표 검증방법 상세화 (II-3장)
- **현상**: 지표 목표치는 있으나 검증 절차/증빙 방법이 불충분
- **작성 방향**:
  - 핵심지표(납기단축률): ERP 주문-배송 완료 시간 로그 → 사업 전 6개월 vs 사업 후 6개월 비교 → 외부 회계법인 검증
  - 특화지표(AI적용률): AI 시스템 커버리지 레이아웃 도면 + 가동 로그 캡처 + 자체 검수보고서
  - 자율지표(MAPE): AI 모델 예측값 vs 실제 배송량 자동 매칭, 월별 롤링 MAPE 대시보드
  - 자율지표(AUROC): 표준 테스트 시나리오 100회 반복, 제3자 검증기관 테스트 성적서

#### 4.9 핵심지표 목표치 상향 검토
- **현상**: 납기 단축률 "10%" → 공고 예시 대비 보수적
- **권장**: **최소 30%** 이상으로 상향 (48시간→34시간, 또는 50%로 24시간 목표)
- **근거**: VMI 도입 시 주문 대기시간 자체가 소멸하므로 30~50% 달성 현실적

#### 4.10 자율지표 3번째 추가 권장
- **현상**: 공고는 "자율지표 2개 이상"이나 현재 딱 2개 (최소선)
- **추가 후보**:
  - **공차율 감소율**: (사업전 30% - 사업후 15%) / 30% × 100 = 50% 감소
  - **위험감지 반응속도**: 300ms 이내 (인간중심 안전보조 테마 자율지표)
- **효과**: 평가위원에게 성과 달성 의지를 강하게 어필

#### 4.11 AI 기능로직 시나리오 가스물류 특화 (II-1장)
- **현상**: 원본의 "AI 기반 불량품 자동 검출 시스템" 예시가 그대로 남아 있음
- **수정 방향**: 가스물류에 맞는 시나리오로 교체
  ```
  기능명       | AI 기반 탱크 잔량 예측 및 VMI 자동보충 시스템
  적용 공정    | 주문→배차→배송 전 프로세스
  기능 목적    | IoT 센서 데이터 기반 고객별 잔량 고갈 시점 예측 및 자동 보충 주문 생성
  적용기술     | LSTM+Prophet 앙상블 시계열 예측, XGBoost 피처 엔지니어링
  입력 데이터  | 탱크 잔량(IoT), 과거 배송이력(ERP), 기상데이터, 산업가동 캘린더
  출력 결과    | 고객별 보충 필요 시점(D+1~D+14), 자동 배송 주문, 긴급도 분류
  운영 시나리오 | ① IoT 센서가 탱크 잔량 68% 수신 → ② AI 모델이 D+1 잔량 20% 예측
               | → ③ 안전재고선 도달 판단 → ④ 자동 보충 주문 생성
               | → ⑤ 배차 최적화 연동 → ⑥ 기사 모바일앱 배송지시
  예상 효과    | 긴급배송 80% 감소, 주문 누락 0%, 고객 이탈률 3%→1%
  ```

#### 4.12 리스크 대응방안 작성
- **현상**: 기술성평가에서 "리스크(돌발변수) 대응방안이 실효성 있게 수립되었는가?" (5점) 평가
