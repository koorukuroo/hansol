# 덕양가스 AI 물류 플랫폼 — 라이브 데모 개발 계획서

> **목적**: 기술성평가 발표(20분) 시 실제 작동하는 데모를 보여줘서 "1년 내 상용화 가능"을 입증
> **핵심 원칙**: 덕양가스 실측 데이터(DINOERP 7,627건)로 작동. 가짜 데이터 없음.
> **기술 스택**: Next.js 14 (App Router) + TypeScript + Tailwind CSS + Recharts + Leaflet Map
> **배포**: Vercel (발표 당일 URL 공유) 또는 로컬 실행

---

## 데모 시나리오 (발표 20분 중 약 8~10분)

발표자가 실제로 클릭하면서 보여주는 흐름:

```
[1] 대시보드 진입 (30초)
 → "이것이 덕양가스 AI 통합 관제센터입니다"
 → 실시간 현황 요약 카드 4개 + 지도에 차량 60대 표시

[2] 수요예측 VMI (2분)
 → 고객 탱크 잔량 리스트 → 위험도 색상 (🔴🟡🟢)
 → "A고객사(현대미포조선) 클릭" → 14일 예측 차트 표시
 → "D+3에 고갈 예측, VMI 자동보충 주문 생성" 버튼 클릭

[3] 배차최적화 (2분)
 → 오늘의 배송 요청 리스트 (VMI + 수동 주문)
 → "AI 최적 배차 실행" 버튼 → 2초 로딩 → 차량별 최적 경로 지도 표시
 → Before(수작업) vs After(AI) 총 거리 비교 표시

[4] 카카오톡 발주 AI (1분)
 → 카톡 메시지 입력란 → "울산대학교 산소 발주 1.4톤" 입력
 → AI가 자동 파싱: 거래처=울산대학교, 품목=산소, 수량=1.4톤
 → "ERP 주문 생성" 버튼 → 주문 목록에 추가

[5] 어라운드뷰 안전 모니터링 (1분)
 → 차량 클릭 → 360° Bird's Eye View 시뮬레이션
 → 보행자 감지 → 경고 레벨 변화 시연 (안전→주의→위험)
 → 이벤트 로그 + 운전자 안전 스코어

[6] 예지보전 (1분)
 → 차량 건강 상태 대시보드 (60대 히트맵)
 → "89보4327 차량 클릭" → ECU 이상 감지 알림
 → RUL 예측: 브레이크 패드 잔여 수명 23일
 → 정비 스케줄 자동 생성

[7] 성과 요약 (30초)
 → KPI 대시보드: 납기단축 30%↑, 공차율 50%↓, 사고 60%↓
 → Before/After 비교 애니메이션
```
