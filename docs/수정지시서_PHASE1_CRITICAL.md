### 현재 문제

### 수정 방안 (택1)

**방안 A (추천): 기업부담금 현금 증액**
- 도입기업 현금 부담금을 **362,500 → 375,000천원**으로 증액 (+12,500천원)

- 기업부담금 비율: 661,640 / 2,207,140 = 30.0% ✅

**방안 B: 정부지원금 감액**
- 정부지원금을 1,545,500 → 1,536,000천원으로 감액 (-9,500천원)
- 감액분을 기업부담금 현금으로 전환
- 새 총사업비: 2,194,690 (동일)
- 정부지원금 비율: 1,536,000 / 2,194,690 = 70.0% ✅
- 기업부담금 비율: 658,690 / 2,194,690 = 30.0% ✅

### B. 핵심지표 상세 수정

**현재 내용을 아래로 교체 (p.17~18 핵심지표 상세표)**:

```
□ 핵심지표: (D) 납기 단축률

  ○ 지표 정의: 고객의 납품 요청일로부터 실제 납품까지 걸린 평균 소요일수를 의미하며, 
    고객이 요청한 납기를 얼마나 빠르게 충족했는지를 나타내는 납기관리 지표

  ○ 측정 산식:
    {(도입 전 납기소요일수) - (도입 후 납기소요일수)} / (도입 전 납기소요일수) × 100
    * 납기소요일수 : (납품일(도착일) - 납기요청일(도착요청일))

  ○ 현재 수준(As-Is): 수작업 배차·주문 처리로 평균 납기소요 ○○일 
    (※ 덕양가스 실측 데이터로 보완 필요)

  ○ 목표 수준(To-Be): 30% 이상 단축 (AI 수요예측 + 동적 배차 통합 효과)

  ○ 도출 근거: 
    - 유사 산업(LPG 물류, 화학물류) AI 배차 도입 사례에서 납기소요 25~35% 단축 실적

  ○ 증빙 방법:
    ① ERP 또는 SCM 시스템에서 추출한 기간별 '납기이행 현황표'

    ② 기간별 제품 수불기록부 또는 고객사 납품 일자를 증명할 수 있는 출하증명서(거래명세서)

```
□ 특화지표: AI 적용률

  ○ 측정 산식:

    - 예지보전: AI 예측보전 적용 차량 / 전체 차량

  ○ 목표 수준: 100% (전체 물류 프로세스에 AI 적용)

  ○ 증빙 방법:
    ① AI 시스템 커버리지를 확인할 수 있는 물류 운영 시스템 아키텍처 도면 또는 구조도
    ② 안전관제플랫폼, MES, ERP 등 시스템 연동 명세서 및 가동 로그 캡처본
    ③ 도입기업 책임자 명의의 원본대조필이 날인된 'AI 적용 현황 검수(자체) 보고서'

### D. 자율지표 상세 수정

```
□ 자율지표 1: 수요·물동량 예측 오차율 (MAPE)

  ○ 지표 정의: AI 수요예측 모델이 고객 탱크의 가스 소비량 및 배송 물동량을 
    예측한 값과 실제 값 간의 평균절대백분율오차

  ○ 측정 산식: MAPE = (1/n) × Σ|실제값 - 예측값| / |실제값| × 100

  ○ 목표 수준: MAPE ≤ 15%

  ○ 도출 근거:
    - 산업용 가스 수요는 계절성·산업가동률에 따른 변동이 크나, IoT 실시간 잔량 데이터와
      기상·가동률 외부 변수를 결합한 LSTM+Prophet 앙상블 모델로 MAPE 12~18% 달성 가능
    - 참고 벤치마크: M5 Forecasting Competition 상위 모델 MAPE 12~15% 수준
    - 초기 학습 데이터 축적 기간을 고려하여 보수적으로 15%로 설정

  ○ 검증 계획:
    - 데이터 분할: 학습(Train) 70% / 검증(Validation) 15% / 평가(Test) 15%
    - 시계열 교차검증: 5-fold Time-Series Cross Validation (시간 순서 보존)
    - 파일럿 검증: 안정화 기간 2개월간 주요 고객 10개소 대상 실측값과 예측값 일일 대조
    - 지속 모니터링: 월별 롤링 MAPE 대시보드 자동 산출, 임계치(20%) 초과 시 모델 재학습

  ○ 미달 시 대응: 앙상블 모델 확대(추가 알고리즘 투입), 피처 엔지니어링 고도화, 
    전문가 보정 루프(Human-in-the-loop) 적용

  ○ 증빙 방법: AI 시스템 예측 로그 vs 실제 배송/소비량 자동 매칭 결과,
    월별 롤링 MAPE 대시보드 캡처, 파일럿 검증 결과보고서

□ 자율지표 2: ETA/납기·지연 예측 정확도 (AUROC)

  ○ 지표 정의: 배송 지연 여부를 사전에 예측하는 이진분류 모델의 판별 성능 지표.
    무작위 분류(0.5) 대비 얼마나 정확하게 지연/정상을 구분하는지 측정

  ○ 측정 산식: AUROC (Area Under the Receiver Operating Characteristic Curve)
    - ROC 곡선 하단 면적, 0.5(무작위)~1.0(완벽) 범위

  ○ 목표 수준: AUROC ≥ 0.85

  ○ 도출 근거:
    - 물류 지연 예측 관련 학술 연구에서 XGBoost/LightGBM 기반 모델로 AUROC 0.82~0.90 달성
    - 본 과제는 IoT 실시간 잔량 + GPS 위치 + 교통 데이터 + 기상 데이터를 결합하므로
      충분한 피처 확보 가능
    - 위험물 운송의 특수성(제한 경로, 충전소 경유)을 반영한 도메인 특화 모델

  ○ 검증 계획:
    - 표준 테스트 시나리오 100회 이상 반복 실행
    - Stratified K-fold Cross Validation (지연/정상 비율 보존)
    - 가능 시 제3자 검증기관(UNIST 등) 테스트 성적서

  ○ 증빙 방법: ROC 곡선 및 AUROC 산출 결과보고서, 테스트셋 혼동행렬(Confusion Matrix),
    시스템 로그 기반 예측-실측 대조표
```

### 삽입 위치: II.1 AI 응용제품 신속 상용화 계획의 마지막 부분 (장비도입 계획 직전, p.11~12 사이)

### 삽입할 내용 전문:

```
□ 제조데이터 표준모델 적용 계획

  ○ 적용 표준: IEC 63278-1 AAS (Asset Administration Shell) for Industrial Applications

  ○ 적용 목적: 
    본 과제에서 수집·활용하는 제조데이터(탱크 잔량, 차량 위치/상태, 배송 이력 등)의 
    표준화를 통해 장비·공정 간 상호운용성을 확보하고, 향후 타 시스템/타 기업과의 
    데이터 연계 기반을 마련함.

  ○ 적용 범위 및 방법:

    (1) AAS 서브모델 정의
      - 가스 탱크 자산: 탱크ID, 위치, 용량, 현재잔량, 소비이력, 센서상태
      - 배송 차량 자산: 차량ID, GPS위치, 적재물정보, ECU상태, 운행이력
      - IoT 센서 자산: 센서ID, 측정항목, 측정주기, 캘리브레이션 정보
      
    (2) 데이터 인터페이스 표준화
      - OPC-UA 통신 프로토콜 적용: AI 엔진 ↔ MES/ERP 데이터 연계 시 
        OPC-UA 기반 표준 인터페이스 구현
      - AAS 표준 속성(Property) 기반 데이터 매핑: 기존 MES/ERP 데이터 필드를 
        AAS 표준 속성으로 매핑하여 표준화된 데이터 딕셔너리 구축

    (3) 도입기업 인프라 적합성
      - 스마트공장 수준: MES + ERP 구축 완료(2022년), 바코드 기반 자산 관리 운영 중
      - 통신 인프라: OPC-UA 부분 구축 상태 → 본 과제를 통해 확장 적용
      - 설비 데이터 수집: 2000년대 이후 생산된 차량/설비로 데이터 수집 가능
      - 조직: IT 담당팀 보유, 데이터 관리 역량 확보

    (4) 기대 효과
      - 장비·공정 간 데이터 상호운용성 확보
      - 향후 동종업계 SaaS 확산 시 표준 데이터 포맷으로 도입 비용 절감
      - 스마트공장 사업관리시스템 연동 로그기록 표준화

  ○ 관련 통신장비 구축: 필요 시 DAQ(Data Acquisition) 장비 및 OPC-UA 게이트웨이를 
    사업비에 계상하여 구축 (N/W 구축비 또는 기타비용으로 편성)
```
