## 프로젝트 개요

- **목적**: 정부 사업 기술성평가 발표(20분) 시 라이브 시연할 웹 데모
- **핵심**: 덕양가스(산업용 가스 물류) 실측 데이터로 작동하는 AI 관제 플랫폼
- **발표일**: 2026년 5월 중 (정확한 일정 추후 확정)
- **데모 시간**: 발표 20분 중 약 8~10분

## 디자인 시스템

### 타이포그래피 스케일

Pretendard 단일 서체. **모든 숫자 표시 영역에 `font-variant-numeric: tabular-nums` 적용.**

```
토큰명          크기    굵기   행간    용도
─────────────────────────────────────────────
display         32px   700    1.15   대시보드 핵심 상태 문장
h1              26px   700    1.2    페이지 제목
h2              20px   600    1.25   섹션 제목
h3              17px   600    1.3    카드 제목
body            15px   400    1.6    일반 텍스트
body-sm         14px   400    1.5    테이블, 리스트 항목
caption         13px   400    1.4    보조 설명, 시간 표시
label           12px   500    1.0    뱃지, 태그, 상태 텍스트
kpi-number      36px   700    1.0    KPI 메인 숫자 (tabular-nums)
kpi-sub         14px   500    1.3    KPI 보조 텍스트
```

텍스트 최대 행 길이: 본문 72자(한글 36자) 이내. `max-w-prose` 또는 `max-w-[640px]`.

### 스페이싱 스케일

8px 베이스. **이 값들만 사용합니다.**

```
토큰    값     용도
────────────────────────────
xs      4px    아이콘-텍스트 간격, 뱃지 내부
sm      8px    카드 내 요소 간격
md      12px   폼 요소 간격
base    16px   카드 패딩, 리스트 항목 간격
lg      24px   섹션 내 그룹 간격
xl      32px   섹션 간 간격
2xl     48px   페이지 상단 여백
```

패널 너비는 비율 기반:
- 좌측 패널: `w-[340px] shrink-0` (고정 최소폭)
- 우측 패널: `flex-1 min-w-0` (나머지 채움)

### 숫자 표기 규칙

```
천 단위 콤마:     1,240km, 7,627건
소수점:          27.7%, 3.2km
단위:            숫자와 단위 사이 공백 없음 (74건, 54대, 1.5억원)
퍼센트:          소수 첫째자리까지 (70.4%)
변화량:          ▲12%, ▼28% (삼각형 + 숫자)
날짜:            YYYY.MM.DD (2026.04.15)
시간:            HH:MM (14:23)
```

### 모션 규격

Framer Motion 사용. **모든 애니메이션에 `prefers-reduced-motion` 대응 필수.**

```
토큰명           지속시간   이징              용도
──────────────────────────────────────────────────
page-enter       250ms     ease-out          페이지 전환 (fadeIn + translateY 8px)
page-exit        150ms     ease-in           페이지 나감
card-hover       200ms     ease-out          카드 호버 (shadow 변화, scale 없음)
gauge-fill       600ms     ease-out          게이지 바 채우기 (200ms 간격 스태거)
count-up         1200ms    ease-out          KPI 숫자 카운트업
toast-enter      300ms     spring(0.5,0.9)   토스트 슬라이드인 (우측 상단)
toast-exit       200ms     ease-in           토스트 페이드아웃
map-route        400ms     ease-in-out       지도 경로 그리기
alert-pulse      1000ms    ease-in-out       위험 알림 펄스 (infinite, 투명도 0.7↔1.0)
```

## 공통 UI 패턴

### 토스트 알림 시스템 (`components/ui/Toast.tsx`)

우측 상단 고정 (top-4 right-4). 최대 3개 스택.

| 유형 | 아이콘 (Lucide) | 좌측 바 색상 | 자동 소멸 |
|------|---------------|------------|---------|
| 성공 | `CheckCircle2` | green-500 | 3초 |
| 경고 | `AlertTriangle` | amber-500 | 5초 (수동 닫기 가능) |
| 에러 | `XCircle` | red-500 | 수동 닫기만 |
| 정보 | `Info` | brand | 3초 |

진입: toast-enter (300ms spring), 퇴장: toast-exit (200ms ease-in).

### 로딩 스켈레톤 (`components/ui/Skeleton.tsx`)

```tsx
// 스켈레톤 — 실제 콘텐츠와 동일한 크기의 회색 블록 + shimmer 애니메이션
<div className="animate-pulse bg-gray-200 rounded-md" style={{ width, height }} />
```

모든 데이터 의존 컴포넌트에 스켈레톤 적용:
- 카드: 카드 크기 그대로, 내부 3줄 스켈레톤
- 차트: 차트 영역 크기 그대로, 수평선 3줄
- 리스트: 행 높이 그대로, 5행 반복
- 지도: 지도 영역에 "지도 불러오는 중..." 중앙 텍스트

### 빈 상태 패턴

모든 리스트/검색 결과에 빈 상태를 구현합니다:

```tsx
// components/ui/EmptyState.tsx
<div className="flex flex-col items-center justify-center py-12 text-gray-400">
  <Icon size={48} strokeWidth={1.5} />        {/* 연한 아이콘 */}
  <p className="mt-3 text-body font-medium text-gray-500">{title}</p>
  <p className="mt-1 text-caption">{description}</p>
  {action && <Button className="mt-4" ...>{actionLabel}</Button>}
</div>
```

| 위치 | 아이콘 | 제목 | 설명 |
|------|--------|------|------|
| 알림 없음 | `Bell` | 알림이 없습니다 | 새로운 이벤트가 발생하면 여기에 표시됩니다 |
| 검색 결과 없음 | `Search` | 검색 결과가 없습니다 | 다른 키워드로 검색해 보세요 |
| 배차 결과 없음 | `Route` | 배송 요청을 선택하세요 | 좌측에서 배송 건을 선택한 후 AI 배차를 실행합니다 |
| 예측 미선택 | `TrendingUp` | 거래처를 선택하세요 | 좌측 리스트에서 거래처를 클릭하면 예측 결과가 표시됩니다 |

### 페이지 전환

사이드바 메뉴 클릭 시 메인 콘텐츠 영역만 전환:
- 나가는 페이지: fadeOut 150ms ease-in
- 들어오는 페이지: fadeIn 250ms ease-out + translateY(8px → 0)
- 사이드바/헤더는 고정 (전환 없음)

## 인터랙션 핵심 포인트 (발표 시 반드시 작동해야 하는 것)

1. **수요예측**: 거래처 클릭 → 스켈레톤(0.3초) → 14일 예측 차트 표시 → "자동보충" 버튼 → 성공 토스트
2. **배차**: "AI 배차 실행" 클릭 → 버튼 내 스피너(2초) → Before/After 지도 경로 표시 (map-route 모션)
3. **카톡**: 메시지 클릭 또는 직접 입력 → 파싱 결과 카드 stagger 진입 → "주문 생성" 성공 토스트
4. **어라운드뷰**: 자동 시연 모드에서 SVG 객체 이동 → 경고 레벨 색상 변화 + 펄스 애니메이션
5. **예지보전**: 히트맵 셀 클릭 → 부품 RUL 게이지 stagger 채우기 (gauge-fill 모션)
6. **전체**: 모든 페이지 전환 시 page-enter/exit 모션 작동

**이 5개가 막힘없이 돌아가면 데모 성공입니다.**
